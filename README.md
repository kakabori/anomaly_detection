# プロジェクト定義
モータ／ファンの異常兆候検知＋原因候補トリアージ
- Phase 1: Agentを載せるための壊れない診断エンジンを作る
- Phase 2: Agentを載せる

## 1. Problem Statement
工場のモータ／ファンについて、振動（1ch）＋温度の時系列データだけを使い、
1. 異常兆候を検知し
2. 異常時に考えられる原因候補を順位付きで提示し
3. なぜそう判断したかを説明可能な形で返す
診断支援ロジックを設計・実装する

## 2. 入力
- 対象：1台のモータ／ファン
- センサ：
  - 振動（1ch）
  - 温度
- データ取得範囲：
  - 診断ロジックが内部で決定する（呼び出し側は意識しない）

## 3. 出力
DiagnosisReport（構造化された診断結果）
以下を含む：
- 異常スコア
- 状態分類
  - NORMAL/WARNING/ANOMALY
- 原因候補 Top 2
  - bearing_wear, imbalance, misalignment, overheating, lubrication issue
- 根拠
  - どの特徴量がどう異常だったか
- データ品質評価
  - 欠損率／ノイズ過多／判断不能フラグ
- 次のアクション提案
  - 点検推奨／継続監視／追加センサ必要

## 4. ルール
- 異常度算出と状態分類ルール
  - 1時間ウィンドウから特徴量を算出
  - 特徴量の24時間トレンドを基に状態分類

## 5. 切り分けロジック
要検討

## 6. Phase 1 ワークフロー
1. 入力検証
2. センサデータ取得（window幅は内部で決定）
3. データ品質評価
4. 特徴量抽出
5. 異常度算出
6. 原因候補生成
7. 根拠付きレポート生成

## 7. Phase 1 の完了条件
- diagnose(machine_id) が動く
- 異常／正常を安定して区別できる
- 原因候補が説明付きで返る
- 型・dataclass・pydanticが使われている
- 失敗理由がログ／例外で追える
  - UIは不要
  - CLI or Python関数でOK
